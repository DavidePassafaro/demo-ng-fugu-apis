File handling is declared in the Web App Manifest via the file_handlers array. This app is already configured — the snippet below reflects the actual manifest.webmanifest.

manifest.webmanifest JSON 
"file_handlers": [
  {
    "action": "/apis/file-handling",
    "accept": {
      "text/plain":       [".txt"],
      "text/markdown":    [".md"],
      "application/json": [".json"],
      "image/png":        [".png"],
      "image/jpeg":       [".jpg", ".jpeg"]
    },
    "icons": [
      { "src": "icons/icon-192x192.png", "sizes": "192x192" }
    ],
    "launch_type": "single-client"
  }
]
  • action — URL the browser navigates to when a registered file is opened. Set to /apis/file-handling so the handler lands on this page.
  • accept — MIME types and extensions this app claims. The OS uses this to show the app in "Open with…" menus.
  • launch_type — "single-client" reuses an existing window instead of opening a new one on each file open.

Manifest changes take effect only after the PWA is reinstalled. If you updated file_handlers on an already-installed app, uninstall and reinstall to pick up the new registrations.
